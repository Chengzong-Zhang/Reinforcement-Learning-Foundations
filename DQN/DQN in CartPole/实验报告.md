# DQN 实验报告

---

## 一、核心机理

### 1.1 问题背景：从表格 Q-learning 到深度强化学习

经典 Q-learning 通过维护一张状态-动作值表 $Q(s,a)$ 来学习最优策略：

$$Q(s,a) \leftarrow Q(s,a) + \alpha \left[ r + \gamma \max_{a'} Q(s',a') - Q(s,a) \right]$$

然而当状态空间连续或维度极高时（如图像输入），表格方法面临"维度诅咒"，无法枚举所有状态。**DQN（Deep Q-Network）** 用一个参数为 $\theta$ 的深度神经网络 $Q(s,a;\theta)$ 来近似值函数，将深度学习的表示能力与 Q-learning 的时序差分（TD）学习规则相结合，是深度强化学习的奠基性工作。

### 1.2 DQN 的两大核心创新

仅将神经网络代入 Q-learning 更新会导致训练发散：网络参数的每次更新都会同时改变当前估值和 TD 目标，形成"追逐不稳定目标"的恶性循环；相邻时间步的样本高度相关，违反随机梯度下降对独立同分布（i.i.d.）样本的假设。DQN 通过两个核心机制解决了这两个问题：

#### 1.2.1 经验回放（Experience Replay）

智能体与环境交互产生的每一条经验 $(s, a, r, s', \text{done})$ 被存入一个固定容量的双端队列——**经验回放池（Replay Buffer）**，容量满后自动丢弃最旧数据。训练时每步从中**随机均匀采样**一个 mini-batch，用于梯度更新。

这一机制的核心价值在于：

- **打破时序相关性**：随机采样使得一个 batch 内的样本来自不同时刻，近似满足 i.i.d. 假设，降低梯度估计方差。
- **提高样本利用率**：每条经验可被多次重复使用，对于采集代价高昂的场景尤为重要。

#### 1.2.2 目标网络（Target Network）

DQN 维护两个结构完全相同的网络：

| 网络 | 参数 | 用途 |
|------|------|------|
| **在线网络（Online Network）** | $\theta$ | 选动作；计算当前 Q 值；接受梯度更新 |
| **目标网络（Target Network）** | $\theta^-$ | 计算 TD 目标，参数定期从在线网络同步（**硬更新**） |

TD 目标为：

$$y = r + \gamma \max_{a'} Q(s', a';\theta^-)$$

损失函数（Bellman 误差的均方误差）：

$$\mathcal{L}(\theta) = \mathbb{E}_{(s,a,r,s') \sim \mathcal{D}} \left[ \left( y - Q(s,a;\theta) \right)^2 \right]$$

目标网络参数 $\theta^-$ 每隔 `target_update` 步才做一次硬更新（$\theta^- \leftarrow \theta$），使 TD 目标在一段时间内保持固定，避免"用不断变化的目标训练"带来的不稳定性。

### 1.3 网络结构

本实验使用的基础 Q 网络为两层全连接 MLP：

```text
输入层 (state_dim)
    → Linear(state_dim, hidden_dim) + ReLU
    → Linear(hidden_dim, action_dim)
输出层 (action_dim 个 Q 值，无激活函数)
```

输出层不加激活函数，使得 Q 值可以是任意实数，与真实价值函数的值域一致。

### 1.4 完整训练流程

```text
初始化：在线网络 θ，目标网络 θ⁻ ← θ，回放池 D
For each episode:
    重置环境，获得初始状态 s
    While not done:
        用 ε-greedy 策略选动作 a
        执行 a，获得 (r, s', done)
        将 (s, a, r, s', done) 存入 D
        If |D| > minimal_size:
            从 D 随机采样 batch B
            计算 TD 目标 y = r + γ·max_a' Q(s',a'; θ⁻)·(1-done)
            最小化 MSE 损失，梯度更新 θ
        If count % target_update == 0:
            θ⁻ ← θ  (硬更新目标网络)
        s ← s'
```

## 二、改进算法

### 2.1 Double DQN：解耦选择与评估

**问题根源**：标准 DQN 的 TD 目标中：

$$y = r + \gamma \max_{a'} Q(s', a'; \theta^-)$$

$\max$ 操作总是选择被估计最高的动作——由于采样噪声，估计最高的 Q 值往往高于真实值，这种"乐观偏差"会被 Bellman 迭代逐步放大，导致**系统性过估计（overestimation bias）**，使策略倾向于高估的次优动作。

**Double DQN 的修正**：将动作的"选择"与"评估"解耦到两个不同的网络：

$$y^{\text{Double}} = r + \gamma \cdot Q\!\left(s',\, \underbrace{\arg\max_{a'} Q(s',a';\theta)}_{\text{在线网络选动作}};\, \underbrace{\theta^-}_{\text{目标网络评估}}\right)$$

由于两个网络的参数不同、偏差方向随机，选出的动作不再必然是被"双重高估"的那个，过估计偏差因此得到有效抑制。

代码实现的核心改动（仅在 `update` 中）：

```python
# Vanilla DQN：目标网络同时做"选"和"估"
max_next_q = target_net(ns).max(1)[0].view(-1, 1)

# Double DQN：在线网络"选"，目标网络"估"
best_a     = q_net(ns).max(1)[1].view(-1, 1)
max_next_q = target_net(ns).gather(1, best_a)
```

### 2.2 Dueling DQN：分流估计状态价值与动作优势

**核心洞察**：Q 值可以分解为状态价值 $V(s)$ 与动作优势 $A(s,a)$ 之和：

$$Q(s,a) = V(s) + A(s,a)$$

其中 $V(s) = \mathbb{E}_a[Q(s,a)]$ 反映"处于当前状态有多好"，$A(s,a) = Q(s,a) - V(s)$ 反映"选当前动作比平均水平好多少"。

**网络结构（VAnet）**：共享特征层之后分为两个独立的输出流：

```text
共享层: Linear(state_dim, hidden_dim) + ReLU
    ├── V 流: Linear(hidden_dim, 1)          → V(s)
    └── A 流: Linear(hidden_dim, action_dim) → A(s,a)

合并: Q(s,a) = V(s) + A(s,a) - mean_a[A(s,a)]
```

减去 $A$ 的均值是为了保证**可识别性**（identifiability）：若不约束，$V$ 和 $A$ 的分解不唯一（可以同时给 $V$ 加常数、$A$ 减常数得到相同 $Q$），减均值后强制 $A$ 的均值为 0，使分解唯一且 $V$ 的学习信号更稳定。

**优势**：在动作间价值差异不大的环境（如 Pendulum），V 流无需区分具体动作即可频繁更新，加速状态价值的学习，整体样本效率更高。

## 五、消融实验（Ablation Study）

### 5.1 实验设计

消融实验在 Pendulum-v1 上逐一移除或替换 DQN 的关键组件，以量化每个设计对最终性能的贡献：

| 变体 | 与基线的差异 | 预期现象 |
|------|------------|---------|
| **VanillaDQN (baseline)** | 完整系统 | — |
| **No Target Network** | TD 目标改用在线网络计算 | "自举自"，目标持续变化，训练震荡 |
| **No Replay Buffer** | 每步用单条经验立即更新（在线学习） | 样本相关性高，梯度方差大，收敛不稳 |
| **Double DQN** | 选择与评估解耦 | 过估计降低，收敛更稳定 |
| **Dueling DQN** | VAnet 双流结构 | 状态价值学习更高效 |
| **Double + Dueling** | 两者叠加 | 综合改进效益 |

### 5.2 学习曲线对比

![消融实验学习曲线](./data%20and%20figures/ablation_curves.png)

曲线显示：去掉目标网络（No Target Network）和去掉回放池（No Replay Buffer）的两条曲线明显低于基线，且波动剧烈；Double DQN 和 Dueling DQN 与基线接近甚至略优；Double + Dueling 在本实验设置下略逊于单独改进，可能存在超参数未充分调优的问题。

### 5.3 最终性能柱状图

![消融实验最终性能](./data%20and%20figures/ablation_bar.png)

### 5.4 数值结果

| 变体 | 最终回报均值（后 20 回合） | 标准差 | 相对基线变化 |
|------|--------------------------|--------|------------|
| VanillaDQN (baseline) | **-241.31** | 115.84 | — |
| No Target Network | -1326.06 | 149.37 | **↓ 449%** |
| No Replay Buffer | -1475.10 | 122.52 | **↓ 511%** |
| Double DQN | -240.44 | 113.56 | ≈ +0.4% |
| Dueling DQN | -240.48 | 110.57 | ≈ +0.3% |
| Double + Dueling | -317.54 | 134.46 | ↓ 31.7% |

### 5.5 Q 值过估计对比（VanillaDQN vs DoubleDQN）

![Q值过估计对比](./data%20and%20figures/q_overestimation.png)

该图直观展示了 Double DQN 对过估计的抑制效果：VanillaDQN 的最大 Q 值曲线整体高于 Double DQN，差距随训练推进而扩大，与理论预期一致——标准 DQN 中 $\max$ 操作的正偏差会被 Bellman 迭代逐步放大。

### 5.6 消融分析结论

1. **目标网络是最关键的稳定性保证**：去掉目标网络后，TD 目标随在线网络同步变化，形成"自举自"的正反馈，性能崩溃至 -1326（降幅 449%）。目标网络通过参数冻结提供了短期稳定的监督信号，是 DQN 能收敛的核心条件。

2. **经验回放是样本效率与收敛稳定性的双重保障**：去掉回放池后，相邻时间步的经验高度相关（同一轨迹），违反 SGD 的 i.i.d. 假设，梯度估计噪声极大，性能降至 -1475（降幅 511%），甚至弱于去掉目标网络的变体。

3. **Double DQN 和 Dueling DQN 是渐进式改进**：在 Pendulum-v1 这一相对简单的任务上，两者对最终收敛性能的提升幅度较小（均不足 1%），但标准差均有所下降，说明改进主要体现在**训练稳定性**而非绝对性能的大幅提升。

4. **Double + Dueling 组合在当前设置下性能略退**：可能原因是两种改进叠加后对学习率等超参数的要求更严格，默认超参数未必是组合版的最优点；也可能是 Pendulum-v1 对这两种改进的需求并不强烈，叠加后引入了额外的训练复杂度。

## 六、超参数敏感性分析（Sensitivity Analysis）

所有敏感性实验均基于 VanillaDQN 在 Pendulum-v1 上进行，每次只改变一个超参数，其余保持默认值（`lr=1e-2, gamma=0.98, epsilon=0.01, target_update=50, batch_size=64`）。

### 6.1 学习率（Learning Rate）

![学习率敏感性](./data%20and%20figures/sensitivity_lr.png)

| lr | 最终回报均值 | 标准差 | 分析 |
|----|------------|--------|------|
| **1e-3** | -1189.90 | 133.74 | 过小：200 回合内几乎无进展，严重欠收敛 |
| **1e-2（默认）** | -242.36 | 122.72 | 适中：收敛快且稳定 |
| **1e-1** | -941.43 | 179.96 | 过大：梯度步长过大，参数震荡，无法稳定收敛 |

**结论**：学习率对性能影响显著且非单调。1e-3 太小导致欠拟合，1e-1 太大导致训练震荡，1e-2 是当前任务下的优选。学习率是 DQN 最敏感的超参数之一，建议在 [1e-3, 1e-2] 范围内进行网格搜索。

## 七、综合结论

1. **经验回放与目标网络是 DQN 成功的两个充分必要条件**：消融实验证明，两者缺一不可。去掉任意一个，回报从约 -241 崩溃至 -1326 至 -1475，性能下降超 4 倍。两者分别解决了"样本相关性"和"训练目标不稳定性"两大根本问题，是 DQN 相比朴素 Q 网络的核心创新。

2. **Double DQN 有效抑制过估计，是值得默认使用的改进**：Q 值过估计分析图直观显示 VanillaDQN 的 Q 值持续高于 Double DQN。虽然在 Pendulum-v1 上最终性能提升有限，但过估计被抑制意味着策略评估更准确，在更复杂任务（如 Atari 游戏）中的优势更为显著。

3. **Dueling DQN 通过结构改进提升样本效率**：V/A 双流分离的核心价值在于 V 流能在不依赖具体动作的情况下频繁更新，加速状态价值的学习。在动作价值差异较小的环境中优势更明显，适合作为默认网络结构替换。

4. **学习率和目标网络更新频率是最关键的调参目标**：两者在敏感性分析中均表现出极高的灵敏度，且错误取值会导致性能断崖式下降（如 `lr=1e-1` 或 `target_update=1`）。调参时应优先确定这两个超参数的合理范围。

5. **较大的 batch_size 在本任务中稳健地提升性能**：batch=256 相比默认 batch=64 将均值从 -242 提升至 -197，方差从 123 降至 86，建议在计算资源允许的情况下适当增大批量大小。
